# Images pour MoneyMirror Landing Page

## Images nécessaires

Pour compléter votre landing page, vous devez ajouter **4 captures d'écran** de votre application MoneyMirror déployée dans ce dossier.

### Liste des images requises:

1. **hero-app.png** (400x400px recommandé)
   - Vue principale de votre application
   - Sera affichée dans la section Hero (en haut)
   - Idéal: capture de la page dashboard avec graphiques visibles

2. **dashboard.png** (800x600px recommandé)
   - Capture d'écran du Dashboard avec les graphiques Chart.js
   - Doit montrer les graphiques de dépenses/revenus
   - Sera affichée dans la première carte de fonctionnalité

3. **transactions.png** (800x600px recommandé)
   - Capture d'écran de la gestion des transactions
   - Doit montrer la liste des transactions avec catégories
   - Sera affichée dans la deuxième carte de fonctionnalité

4. **savings.png** (800x600px recommandé)
   - Capture d'écran de la fonction Piggy Bank (objectifs d'épargne)
   - Doit montrer les objectifs d'épargne et leur progression
   - Sera affichée dans la troisième carte de fonctionnalité

## Comment prendre les screenshots

### Étape 1: Ouvrir votre application
1. Visitez: https://holbertonschool-money-mirror.vercel.app/
2. Connectez-vous si nécessaire
3. Assurez-vous d'avoir des données de démonstration visibles

### Étape 2: Prendre les captures d'écran

#### Sur Mac:
- **Capture d'écran complète**: `Cmd + Shift + 3`
- **Capture d'écran sélective**: `Cmd + Shift + 4` (puis sélectionnez la zone)
- **Capture d'une fenêtre**: `Cmd + Shift + 4`, puis appuyez sur `Espace`, puis cliquez sur la fenêtre

#### Sur Windows:
- **Outil Capture**: Rechercher "Outil Capture" dans le menu Démarrer
- **Capture plein écran**: `Win + Print Screen`
- **Capture sélective**: `Win + Shift + S`

#### Sur Linux:
- Utilisez `gnome-screenshot` ou l'outil de capture de votre environnement de bureau

### Étape 3: Optimiser les images

1. **Rogner** les images pour ne garder que la partie importante de l'application
2. **Redimensionner** si nécessaire (les images trop grandes ralentissent le chargement)
3. **Renommer** exactement comme indiqué ci-dessus
4. **Placer** les images dans ce dossier (`images/`)

## Formats acceptés

- PNG (recommandé pour la qualité)
- JPG/JPEG (plus léger, bon pour le web)
- WebP (optimal pour les performances web)

## Conseils pour de bonnes captures

- Utilisez un navigateur en mode normal (pas de mode développeur ouvert)
- Assurez-vous que les données affichées sont significatives et représentatives
- Évitez les données personnelles sensibles si vous rendez le site public
- Privilégiez des captures claires et bien éclairées
- Assurez-vous que le texte est lisible

## Alternative: Créer des mockups

Si vous n'avez pas encore de données dans votre app, vous pouvez:
1. Créer des données de test dans votre application
2. Utiliser des outils de mockup comme Figma ou Canva
3. Utiliser des captures de votre environnement de développement local

## Vérification

Après avoir ajouté les images, ouvrez `index.html` dans votre navigateur pour vérifier qu'elles s'affichent correctement.
